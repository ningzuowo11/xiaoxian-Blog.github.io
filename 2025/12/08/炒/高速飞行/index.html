完整指南：从购买代理云主机到设置全局代理（2025年12月实测，10-15分钟搞定）

阶段一：购买代理云主机（青果网络，推荐入门级配置）

青果网络（https://www.qg.net）是 2025 年最亲民的 VPS 平台，支持 CN2 GIA 线路，月付低至 50 元起，1 核 1G 够用。

  1. 注册账号

  2. 进入云计算页面
    登录后 → 左侧菜单「云计算」 → 「云主机」 → 「购买云主机」。

  3. 选择配置（推荐入门级，够用不浪费）

    • 地区：亚洲 → 日本 → 东京（CN2 GIA 线路，延迟 50-80ms）
    • 配置：
      项目 推荐选择 价格（月付） 说明
      CPU 1 核 - 够代理用
      内存 1G - 基础够
      硬盘 20G SSD - 够系统+面板
      带宽 1M (1000Mbps 峰值) ≈50-80元/月 跑满 100M 带宽无压力
      系统 Ubuntu 20.04 / AlmaLinux 8 - Linux 最好（别选 Windows）

  4. 等待创建（2-5 分钟）
    支付成功后 → 返回「云主机列表」 → 刷新页面，等状态从「创建中」变「运行中」。
    关键信息（记下来）：

    • 公网 IP：如 45.251.137.159（创建完自动显示）
    • 业务标识：z68ga5r1（你的订单号）
    • root 密码：重装系统时自己设置（默认随机，记牢）。

提示：如果 IP 没出来，点「刷新」或等 5 分钟。创建完发邮件通知。

阶段二：远程连接服务器（用 PuTTY，1 分钟配置）

服务器创建好后，用 SSH 连接（端口 22）管理它。推荐 PuTTY（免费、稳定）。

  1. 下载 PuTTY
    直链：https://the.earth.li/~sgtatham/putty/latest/w64/putty.exe（便携版，1MB，无需安装）。
    下载后扔到桌面，重命名为 qg-putty.exe。

  2. 青果面板一键配置 PuTTY（可选，但超方便）
    青果控制台 → 云主机列表 → 你这台服务器 → 「远程连接」 → 选 PuTTY → 浏览选 putty.exe → 保存。
    以后点「一键 SSH」自动打开 PuTTY + 填好 IP。

  3. 手动配置 PuTTY 会话（永久保存，一键连接）
    双击 putty.exe → 按下面填（左侧菜单顺序）：

    分类 项目 填写内容
    Session Host Name 你的公网 IP（如 45.251.137.159）
    Port 22
    Connection type SSH
    Connection → Data Auto-login username root
    Window → Translation Remote character set UTF-8
    Session (底部) Saved Sessions 输入名字如「东京GIA」 → Save

    点 Open → 弹安全警告 → 「是」 → 输入 root 密码 → 成功登录（黑屏命令行）。

提示：首次连接慢点正常，以后 0.3 秒进。升级：生成 Ed25519 密钥免密码（可选，SSH 菜单生成公钥 → 复制到服务器 ~/.ssh/authorized_keys）。

阶段四：添加节点 + 客户端使用（5 分钟起飞）

登录面板（http 先进，SSL 后用 https）。

  1. 添加最强 Reality 节点
    左侧「入站列表」 → 「添加入站」 → 填表（其他默认）：

    项目 填写内容 说明
    备注 日本GIA-Reality
    协议 VLESS
    端口 443
    传输协议 TCP (RAW) 必须 RAW 才出 Flow
    Flow xtls-rprx-vision 核心选项
    安全 Reality
    Fingerprint chrome
    Target/dest www.microsoft.com:443
    SNI www.microsoft.com
    Short IDs 0123456789abcdef
    SpiderX /

    点「创建」 → 成功！

  2. 客户端导入
    点节点右边二维码/订阅链接 → 导入客户端：

    • Windows/Mac：Nekoray / Clash Verge Rev
    • Android：NekoBox / v2rayNG
    • iOS：Shadowrocket / Stash
      更新订阅 → 选节点 → 全局模式 → 测试：延迟 45-65ms，YouTube 8K 秒开。

日常维护 + 故障排除

  • 命令：x-ui restart（重启）、x-ui status（状态）、x-ui log（日志）
  • 常见问题：
    问题 解决
    面板进不去 用 http 先，改 SSL 后用 https
    Flow 不显示 选 TCP (RAW) 再选安全 Reality
    节点连不上 x-ui restart + 检查防火墙
    被封 IP 换端口/协议，或买新 IP

阿里云域名解析与真实 TLS 证书配置

虽然 Reality 协议可以借用别人的证书（如 microsoft.com），但如果你希望让后台面板实现专属域名安全访问，或者使用如 VLESS-WSS (WebSocket + TLS)、Trojan 等更具伪装性的标准安全协议，你就需要将你在 阿里云 购买的域名解析到青果网络 VPS，并申请专属的 TLS 证书。

1. 在阿里云配置域名解析（连接域名与服务器 IP）

假设你在阿里云购买的主域名为 yourdomain.com，现在要将它与你的服务器公网 IP（如 45.251.137.159）绑定：

  • 登录阿里云控制台：
    访问阿里云官网，登录后进入右上角的「控制台」，搜索并进入 「云解析 DNS」。
  • 进入域名解析设置：
    在域名列表中找到你的域名，点击右侧的 「解析设置」。
  • 添加解析记录：
    点击 「添加记录」 按钮，在弹出的表单中精确填写以下参数：
    • 记录类型：选择 A （表示将域名指向一个 IPv4 地址）。
    • 主机记录：填写你自定义的二级域名前缀。例如填写 vps，那么你的完整节点域名就是 vps.yourdomain.com；如果填 @，则直接使用主域名。（推荐填写 vps 或其他短字母用于代理专用）。
    • 解析线路：保持 默认 即可。
    • 记录值：粘贴你在阶段一获得的 服务器公网 IP（如 45.251.137.159）。
    • TTL：保持默认的 10分钟。
  • 保存并检查生效状态：
    点击「确认」保存。通常 1 分钟左右解析就会在全网生效。
    • 验证方法：在本地电脑打开终端（Windows 的 cmd 或 Mac 的 Terminal），输入 ping vps.yourdomain.com（替换为你自己的域名）。若返回的 IP 为你的 VPS IP，说明解析成功。

3. 在 3X-UI 面板中配置域名与 TLS 证书

证书申请成功后，我们需要在面板中进行最后的对接调用。

通过专属域名加密访问 3X-UI 后台面板
  1. 此时先通过原 IP 访问面板：http://45.251.137.159:54321/j8k7h6g5f4d3s2a1。
  2. 登录后点击左侧菜单的 「面板设置」 (Panel Settings)。
  3. 找到配置项，分别填入你刚刚生成的证书绝对路径：
  • **面板证书公钥路径 (Cert file path)**：/etc/x-ui/certs/cert.crt
  • **面板证书私钥路径 (Key file path)**：/etc/x-ui/certs/private.key

  1. 点击 「保存设置」，然后在 PuTTY 终端输入 x-ui restart 重启面板服务使其生效。

  2. 安全访问：今后，你必须在浏览器输入完整的包含 HTTPS 和域名的地址 https://vps.yourdomain.com:54321/j8k7h6g5f4d3s2a1 才能安全登录面板（此时地址栏会亮起安全锁，直接用原 IP 访问将被彻底阻断，极大提升了防主动探测能力）。

  3. 设定完成后，点击 「添加」，生成新的节点供客户端导入使用。
